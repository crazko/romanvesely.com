It's possible to protect against [CSRF attack](https://owasp.org/www-community/attacks/csrf) without traditional hidden input fields with tokens.
Header [`Sec-Fetch-Site`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Sec-Fetch-Site) is sent by browsers with every request since 2023.
This [header should be read by server](https://blog.miguelgrinberg.com/post/csrf-protection-without-tokens-or-hidden-form-fields) that can act according to its value.
For example, reject requests that are not `same-origin`.
